Visual Studio Code C/C++

Visual Studio Code C/CPP配置指南

一.准备工作(仅Windows)

1.安装MinGW编译器

在官网上下载MinGW编译器，并进行环境变量的配置。

win键+s 搜索环境变量，点击系统变量，PATH，编辑，点击新建，然后点击浏览选择MinGW安装路径(选择到bin目录处)

2.安装Visual Studio Code的C/CPP插件

直接在Visual Studio Code搜索C/C++ Extension Pack(C/CPP扩展包)安装即可，安装Code Runer插件

二.Visual Studio Code配置工作

切记在配置运行环境时，请始终保持Visual Studio Code将C/CPP源文件在编辑器里处于编辑状态，这样Visual Studio Code会根据打开的源文件类型智能推荐配置工作和配置文件中编译程序路径的自动补全。

1.创建C/CPP源文件

创建任意C/CPP源文件，并键入HelloWorld代码

2.配置生成任务

点击终端->配置默认生成任务，然后根据编译器提供的选择生成程序，在MinGW中C使用GCC,Cpp使用g++，vscode会自动生成task.json文件(无需修改)

点击菜单栏的终端，运行生成任务(Ctrl+F9)，将C/C++文件编译为可执行文件，若有错误根据报错进行修改。必须先执行生成任务后，才可进行调试！

3.配置调试文件

保持C/Cpp源文件处于编辑状态，点击运行->添加配置，选择C/C++(GDB/LLDB)，然后选择调试程序(C选择GCC,C++选择G++),然后Visual Studio Code会自动生成launch.json文件。

此处可能会有三种情况发生
1.生成完全版本launch.json，即configurations内容已经完全补充完整
2.生成非完全版本launch.json，configuration内容为空，点击添加配置，选择(gdb) 启动，替换program内容为task.json的program，miDebuggerPath选择gdb所在文件夹路径(常用于自定义生成目录)
3.无需launch.json文件，打上断点后直接按F5即可运行(目前常见，安装Code Runer插件可一键运行和调试)
